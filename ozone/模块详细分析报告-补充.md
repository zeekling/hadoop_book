# 第一部分：Storage Container Manager (server-scm) 详细分析

## 一、核心功能描述

**Storage Container Manager (SCM)** 是Apache Ozone分布式存储系统的核心组件，负责管理和协调整个集群中的数据存储资源。SCM模块是Ozone架构中的关键中枢组件，承担着以下核心职责：

### 1.1 容器管理 (Container Management)

SCM负责容器的创建、分配、状态管理和生命周期控制。容器是Ozone中数据存储的基本单元，SCM通过`ContainerManager`接口及其实现`ContainerManagerImpl`来管理所有容器。容器具有多种状态（ALLOCATED, OPEN, CLOSING, CLOSED, QUASI_CLOSED, DELETING, DELETED），SCM通过状态机来控制容器的状态转换。

### 1.2 管道管理 (Pipeline Management)

SCM使用管道（Pipeline）来组织多个Datanode形成数据复制通道。通过`PipelineManager`接口及其实现`PipelineManagerImpl`，SCM可以创建、打开、关闭和删除管道。管道支持两种类型：RATIS管道（基于Ratis共识协议）和EC管道（纠删码管道）。管道管理确保数据能够在多个Datanode之间高效传输和复制。

### 1.3 节点管理 (Node Management)

通过`NodeManager`接口及其实现`SCMNodeManager`，SCM管理整个集群中的所有Datanode。节点管理包括：
- 节点注册和心跳处理
- 节点状态跟踪（HEALTHY, STALE, DEAD, DECOMMISSIONED）
- 节点操作状态管理（IN_SERVICE, DECOMMISSIONING, DECOMMISSIONED, MAINTAINANCE）
- 节点资源使用监控

### 1.4 复制管理 (Replication Management)

`ReplicationManager`负责确保容器的副本数量符合预期配置。它监控容器副本状态，检测不足副本或多余副本，并触发相应的复制或删除操作。复制管理支持多种复制类型：RATIS（3副本）、EC（纠删码如6+3）等。

### 1.5 安全和证书管理

SCM包含完整的安全基础设施，包括：
- X.509证书管理（通过`SCMCertStore`）
- 根CA和子CA管理
- 证书轮换（通过`RootCARotationManager`）
- 密钥管理（通过`SecretKeyManagerService`）

### 1.6 高可用性 (High Availability)

SCM支持基于Ratis的HA架构，通过`SCMHAManager`接口及其实现`SCMHAManagerImpl`来管理HA。HA配置允许部署多个SCM实例，通过Ratis协议实现状态同步和领导者选举。

### 1.7 安全模式 (Safe Mode)

`SCMSafeModeManager`管理SCM的安全模式，确保在集群达到健康状态之前不会处理客户端请求。安全模式规则包括：
- DataNodeSafeModeRule：数据节点数量
- OneReplicaPipelineSafeModeRule：至少一个开放管道
- RatisContainerSafeModeRule：RATIS容器数量
- ECContainerSafeModeRule：EC容器数量
- HealthyPipelineSafeModeRule：健康管道数量

# 第二部分：Ozone Manager 详细分析

## 一、核心功能详细描述

Ozone-Manager（简称OM）是Apache Ozone分布式存储系统的元数据管理核心组件，负责整个Ozone命名空间的管理工作。作为Ozone架构中的关键组件，OzoneManager扮演着类似于HDFS中NameNode的角色，但进行了大量现代化设计以支持大规模分布式存储场景。

该模块的核心职责包括：卷（Volume）管理、桶（Bucket）管理、键（Key）管理、多租户支持、快照管理、ACL权限控制以及与Storage Container Manager（SCM）的高度集成。

从架构层面来看，OzoneManager采用基于Ratis的强一致性复制协议来实现高可用性（HA）部署。在HA模式下，多个OzoneManager实例通过Raft共识算法保持元数据的一致性，确保即使某个节点发生故障，其他节点也能继续提供服务。

OzoneManager的元数据存储采用RocksDB嵌入式数据库，这是一种高性能的键值存储系统，专为SSD优化。每个OzoneManager实例维护着一个本地RocksDB数据库，其中存储了所有卷、桶、键的元信息以及多租户相关的数据。

在多租户支持方面，OzoneManager实现了完整的租户隔离机制。每个租户拥有独立的卷空间，租户之间的数据完全隔离。同时，OzoneManager还集成了Ranger授权服务，支持细粒度的访问控制策略。

此外，OzoneManager还负责后台维护任务的调度和执行，包括：
- 键删除服务（KeyDeletingService）
- 目录删除服务（DirectoryDeletingService）
- 快照删除服务（SnapshotDeletingService）
- SST文件过滤服务（SstFilteringService）

# 第三部分：S3Gateway 详细分析

## 一、核心功能描述

Apache Ozone 的 S3Gateway 模块是 Ozone 对象存储系统的 S3 兼容 REST API 网关，使得原生支持 AWS S3 API 的应用程序能够无缝访问 Ozone 对象存储。

该模块的核心设计目标是为 Ozone 提供完全兼容 Amazon S3 RESTful API 规范的接口层，从而实现与现有 S3 生态系统工具和 SDK（如 AWS CLI、boto3、Java S3 SDK 等）的即插即用兼容性。

从架构角度来看，S3Gateway 基于 JAX-RS（Java API for RESTful Web Services）框架构建，使用 Jersey 作为默认的 REST 实现，运行在嵌入式 Jetty HTTP 服务器之上。

认证机制方面，S3Gateway 支持 AWS Signature Version 2 和 Version 4 两种签名算法，这是实现 S3 兼容性的核心组件，能够验证客户端请求的完整性和来源。

S3Gateway 支持的关键功能包括：
- 根级别：ListBuckets 操作
- Bucket 级别：CreateBucket、DeleteBucket、HeadBucket、ListObjects 等
- Object 级别：PutObject、GetObject、HeadObject、DeleteObject、CopyObject 等
- 分段上传：完整的生命周期管理
- ACL 和标签管理

# 总结

本文档详细分析了 Apache Ozone 的三个核心模块：

1. **Storage Container Manager (SCM)** - 负责整个集群的元数据、容器、管道和节点管理
2. **Ozone Manager (OM)** - 负责卷、桶、键的元数据管理，支持HA和多租户
3. **S3Gateway** - 提供S3兼容的REST API，使AWS S3客户端可以访问Ozone

这些模块共同构成了Ozone分布式存储系统的基础架构，为企业级大规模存储提供了可靠的支持。
