# Apache Ozone 模块分析

## 项目整体结构

Ozone是一个三层Maven多模块项目：

```
ozone-main (根)
├── dev-support       # 开发支持工具
├── hadoop-hdds       # Hadoop Distributed Data Store (底层存储)
└── hadoop-ozone      # Ozone核心 (上层应用)
```

版本：2.2.0-SNAPSHOT (Katmai)

## 模块依赖关系

```
hadoop-hdds (底层存储)
    │
    ├── client              # 客户端库
    ├── common              # 通用组件
    ├── config              # 配置
    ├── container-service   # 容器服务
    ├── erasurecode         # 纠删码
    ├── interface-*         # 接口定义
    ├── managed-rocksdb     # RocksDB
    ├── rocks-native        # 原生库
    └── server-scm          # SCM
           │
           ▼
hadoop-ozone (上层应用)
    │
    ├── ozone-manager       # 元数据管理
    ├── s3gateway           # S3协议
    ├── ozonefs             # 文件系统
    ├── datanode            # 数据节点
    ├── recon               # 监控
    └── dist                # 最终分发
```

## 关键组件职责

### SCM (Storage Container Manager)
- 容器生命周期管理
- 块分配和副本管理
- 集群拓扑管理

### OM (Ozone Manager)
- 卷（Volume）管理
- 桶（Bucket）管理
- 键（Key）元数据管理

### Datanode
- 实际数据存储
- 容器数据读写
- 块报告和心跳

### S3Gateway
- S3协议兼容
- 请求路由

## 总结

| 层级 | 包含模块 |
|------|----------|
| 根 | dev-support |
| 底层存储 | hadoop-hdds (19模块) |
| 上层应用 | hadoop-ozone (27+模块) |

### 设计特点
1. **分层架构**：HDDS提供底层存储，Ozone提供对象存储服务
2. **多协议支持**：S3、HDFS、CSI等
3. **模块化设计**：清晰的职责分离
4. **条件编译**：根据JDK版本和构建选项动态包含模块
5. **完整的测试体系**：单元测试、集成测试、故障注入测试
