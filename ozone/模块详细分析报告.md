# Apache Ozone 完整模块详细分析报告

本报告详细分析 Apache Ozone 项目（版本 2.2.0-SNAPSHOT）的所有模块，按照功能分组进行深入分析。

---

# 第一部分：hadoop-hdds 存储层模块

## 一、核心存储层

### 1. server-scm (Storage Container Manager)

**核心功能描述**

Storage Container Manager（SCM）是Ozone架构中的核心组件，负责管理整个集群的元数据、容器生命周期、节点管理和管道（Pipeline）分配。SCM作为集群的"大脑"，协调DataNode之间的通信，管理容器的创建、删除和复制，并提供高可用性支持。

**关键类/接口**

| 类型 | 名称 | 说明 |
|------|------|------|
| 核心类 | `StorageContainerManager` | SCM主入口类，管理所有SCM服务 |
| 核心类 | `StorageContainerManagerStarter` | SCM启动器 |
| 核心类 | `StorageContainerManagerHttpServer` | HTTP服务器，提供JMX和REST接口 |
| 核心接口 | `NodeManager` | 节点管理接口，负责DataNode注册和心跳 |
| 核心接口 | `ContainerManager` | 容器管理接口，负责容器生命周期 |
| 核心接口 | `PipelineManager` | 管道管理接口，负责数据复制管道 |
| 核心接口 | `BlockManager` | 块管理接口，负责块分配和删除 |

**主要方法**

```
StorageContainerManager:
├── start() - 启动SCM服务
├── stop() - 停止SCM服务
├── getContainerManager() - 获取容器管理器
├── getPipelineManager() - 获取管道管理器
├── getNodeManager() - 获取节点管理器
└── getBlockManager() - 获取块管理器

NodeManager:
├── register(DatanodeDetails) - 注册DataNode
├── getNodes(NodeState) - 获取指定状态的节点列表
├── handleHeartbeat(DatanodeDetails) - 处理节点心跳
└── getNodeDatanodeDetails() - 获取节点详情

ContainerManager:
├── createContainer(ContainerID, ReplicationConfig) - 创建容器
├── getContainer(ContainerID) - 获取容器信息
├── updateContainerState(ContainerID, LifeCycleEvent) - 更新容器状态
└── deleteContainer(ContainerID, boolean) - 删除容器
```

### 2. container-service

**核心功能描述**

Container-service模块是Ozone在DataNode端的容器服务实现，负责实际的数据存储操作。该模块处理容器的I/O请求、块数据读写、容器复制、扫描和健康检查等核心存储功能。

| 核心类 | `KeyValueContainer` | Key-Value类型容器实现 |
| 核心接口 | `Container` | 容器操作接口 |
| 核心接口 | `ContainerDispatcher` | 容器请求分发器 |
| 核心接口 | `Handler` | 容器请求处理器 |
| 核心接口 | `ChunkManager` | 块数据管理接口 |
| 核心接口 | `BlockManager` | 块管理接口 |

**主要方法**

```
OzoneContainer:
├── OzoneContainer() - 构造函数，初始化容器
├── start() - 启动容器服务
├── stop() - 停止容器服务
├── getContainerSet() - 获取容器集
├── getWriteChannel() - 获取写通道(Ratis)
```

### 3. erasurecode

**核心功能描述**

Erasure-code（纠删码）模块提供高性能的EC编码/解码实现，支持Reed-Solomon（RS）和XOR两种算法。该模块实现了低级的原始纠删码编解码器，可用于EC复制的分布式存储，提高存储效率并降低成本。

**关键类/接口**

| 类型 | 名称 | 说明 |
|------|------|------|
| 核心类 | `RawErasureEncoder` | 原始编码器抽象类 |
| 核心类 | `RawErasureDecoder` | 原始解码器抽象类 |
| 核心类 | `RSRawEncoder` | RS编码器实现 |
| 核心类 | `RSRawDecoder` | RS解码器实现 |
| 核心类 | `XORRawEncoder` | XOR编码器实现 |
| 核心类 | `XORRawDecoder` | XOR解码器实现 |
| 核心类 | `ErasureCodeNative` | 本地EC库包装类 |
| 核心接口 | `RawErasureCoderFactory` | 编码器工厂接口 |

**主要方法**

```
RawErasureEncoder:
├── encode(ByteBuffer[], ByteBuffer[]) - 编码数据
├── doEncode(ByteBufferEncodingState) - 执行编码
└── encode(ByteBuffer[], ByteBuffer[]) - 数组编码

RawErasureDecoder:
├── decode(ByteBuffer[], int[], ByteBuffer[]) - 解码数据
├── doDecode(ByteBufferDecodingState) - 执行解码
└── decode(ByteBuffer[], int[], ByteBuffer[]) - 数组解码

ErasureCodeNative:
├── loadNative() - 加载本地库
├── isNativeCodeLoaded() - 检查本地库加载状态
└── getLibraryName() - 获取库名称
```

### 4. managed-rocksdb

**核心功能描述**

Managed-rocksdb模块提供了RocksDB数据库的托管封装，包装了原生RocksDB API并添加了资源跟踪、内存管理和泄漏检测功能。该模块确保RocksDB资源能够被正确关闭，防止内存泄漏。

**关键类/接口**

| 类型 | 名称 | 说明 |
|------|------|------|
| 核心类 | `ManagedRocksDB` | RocksDB托管封装类 |

| 核心类 | `ManagedDBOptions` | 数据库选项托管 |
| 核心类 | `ManagedColumnFamilyOptions` | 列族选项托管 |
| 核心类 | `ManagedWriteOptions` | 写选项托管 |
| 核心类 | `ManagedRocksIterator` | 迭代器托管 |
| 核心接口 | `ManagedObject` | 托管对象基接口 |

**主要方法**

```
ManagedRocksDB:
├── open(DBOptions, String, List<ColumnFamilyDescriptor>, List<ColumnFamilyHandle>) - 打开数据库
├── openReadOnly(ManagedOptions, String) - 只读打开
├── close() - 关闭数据库
├── write(WriteOptions, WriteBatch) - 写数据
├── get(byte[]) - 读取数据
├── put(byte[], byte[]) - 写入键值对
├── delete(byte[]) - 删除键值对
├── newIterator() - 创建迭代器
└── getProperty(String) - 获取属性

ManagedObject:
├── get() - 获取原始对象

```

### 5. rocks-native

**核心功能描述**

Rocks-native模块负责加载和管理RocksDB的原生工具库（rocksdb-tools），包括SST文件迭代器等原生功能。该模块支持直接从SST文件中读取数据，无需打开完整的数据库实例，提高了大数据量场景下的性能。

**关键类/接口**

| 核心类 | `NativeConstants` | 原生库常量定义 |
| 核心类 | `NativeLibraryNotLoadedException` | 加载异常类 |

**主要方法**

```
NativeLibraryLoader:

├── getJniLibraryFileName() - 获取JNI库文件名
├── isMac() / isWindows() / isLinux() - 操作系统检测
└── extractLibrary(String, Path) - 提取库文件

ManagedRawSSTFileReader:
├── open(String) - 打开SST文件
├── newIterator() - 创建迭代器
├── getProperty(String) - 获取属性
```

## 二、接口和客户端层

### 9. interface-client

**核心功能描述**

定义了客户端与DataNode之间通信的Protocol Buffer接口规范。这是Ozone分布式存储系统的底层客户端接口，包含了容器操作、块操作、Chunk操作等核心协议定义。

**关键类和接口**

| 类型 | 名称 | 说明 |
|------|------|------|
| Proto文件 | `DatanodeClientProtocol.proto` | 客户端到DataNode的RPC协议定义 |
| Proto文件 | `hdds.proto` | 通用数据结构定义（Pipeline、Container、Datanode等） |
| 生成的Java类 | `ContainerProtos` | 容器命令请求/响应消息 |
| 生成的Java类 | `HddsProtos` | 通用协议消息和枚举 |
| gRPC服务 | `XceiverClientProtocolService` | 流式客户端-DataNode RPC服务 |
| gRPC服务 | `IntraDatanodeProtocolService` | DataNode间容器复制服务 |

**主要方法/接口**

- **容器操作**: CreateContainer, ReadContainer, UpdateContainer, DeleteContainer, CloseContainer
- **块操作**: PutBlock, GetBlock, DeleteBlock, ListBlock, GetCommittedBlockLength
- **Chunk操作**: ReadChunk, WriteChunk, DeleteChunk, ListChunk
- **小文件操作**: PutSmallFile, GetSmallFile (合并块和Chunk操作)
- **流操作**: StreamInit, StreamWrite, FinalizeBlock

---

### 15. Ozone Manager (ozone-manager)

**核心功能描述**

Ozone Manager 是 Ozone 的**元数据管理器**，负责管理整个 Ozone 存储系统的元数据。它管理卷（Volume）、桶（Bucket）和键（Key）的元数据信息，并提供 RPC 接口供客户端调用。

**关键类/接口**

| 类/接口 | 说明 |
|---------|------|
| `OzoneManager` | 主类，实现 OzoneManagerProtocol 和 OMInterServiceProtocol |
| `OzoneManagerProtocol` | 客户端协议接口 |
| `VolumeManager` | 卷管理接口 |
| `BucketManager` | 桶管理接口 |
| `KeyManager` | 键管理接口 |
| `OMMetadataManager` | 元数据存储管理 |

**主要方法**

```java
// 卷操作
createVolume(String volumeName, OmVolumeArgs args)
deleteVolume(String volumeName)
lookupVolume(String volumeName)
listVolume(String userName, String prefix, String prevKey, int maxKeys)
```

### 16. S3 Gateway (s3gateway)

**核心功能描述**

**主要API端点**

```java
// 桶操作
```

# 总结

Apache Ozone 项目包含 40+ 个模块，采用分层架构设计：

| 层级 | 主要模块 |
|------|----------|
| **底层存储 (HDDS)** | server-scm, container-service, datanode, erasurecode, managed-rocksdb |
| **接口定义** | interface-client, interface-server, interface-admin |
| **核心服务** | ozone-manager, s3gateway, httpfsgateway, datanode |
| **文件系统** | ozonefs, ozonefs-common, ozonefs-hadoop2, ozonefs-hadoop3 |
| **安全和多租户** | multitenancy-ranger, s3-secret-store, crypto-api, crypto-default |
| **运维工具** | recon, insight, freon, cli-*, tools, csi |
| **测试** | integration-test, mini-cluster, fault-injection-test |
| **分发** | dist |

这种模块化设计使得 Ozone 能够支持多种协议（S3、HDFS、CSI），提供企业级的安全和多租户功能，并保持良好的可扩展性和可维护性。
