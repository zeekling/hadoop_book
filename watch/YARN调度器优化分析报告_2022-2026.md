### 🥈 主题 2：并发调度器建设（架构级改造）

这是 YARN 调度器领域最重大的架构变更。

| JIRA ID | 标题 | 状态 | 持续 | 说明 |
|:--------|:-----|:-----|:----|:------|
| **YARN-11411** | [Umbrella] 构建并发 YARN 调度器 | **In Progress** | 3.5年+ | 顶层架构设计 |
| **YARN-11440** | 创建并发 Leaf Queue | **Open** | 3.5年+ | 核心数据结构 |
| **YARN-11412** | 创建并发 Users Manager | **In Progress** | 3.5年+ | 用户管理并发化 |
| YARN-11693 | 重构 Container 调度器 | **Open** | 2年+ | 容器分配重构 |

**分析**：并发调度器是解决大集群调度延迟的根本方案。当前 YARN 调度器使用全局写锁，在数千节点、数万容器场景下成为瓶颈。但此项目已推进 3.5 年尚未完成，表明复杂度极高。

## 五、社区动态总结

| 维度 | 发现 |
|:-----|:------|
| **最活跃领域** | 性能优化（~15 个 Issue）> CS UI（~14 个）> 抢占（~8 个）|
| **修复率** | 整体约 **50%**，调度正确性类修复率最高（90%），CS 功能类最低（~30%）|
| **重大发布** | **3.4.0**: 2022-2023 基础修复集中发布；**3.5.0**: Global Scheduler 加速 + CS UI + 过量分配修复 |
| **架构级项目** | 并发调度器（YARN-11411）已推进 3.5 年仍未完成，是最大技术债 |
| **新方向** | CS UI 是 2025-2026 最大投入（14 个子任务），标志着从功能到可观测性的转变 |
| **性能趋势** | 从 2022 的配置加载优化 → 2024 的调度路径匹配优化 → 2025 的核心容器分配加速 |
| **饥饿问题** | 2026 年 Fair Scheduler 的饥饿问题（YARN-11907, 11968, 11966）集中涌现，值得关注 |

### 核心结论

1. **3.5.0 是 YARN 调度器优化的里程碑版本** — YARN-11752（分配加速）、YARN-11798（预检查分离）、YARN-11702（过量分配修复）、YARN-11834（ACCEPTED 卡住修复）、YARN-11885（CS UI）均在 3.5.0 合并
2. **并发调度器是最大的未完成工程** — YARN-11411 已推进 3.5 年，是整个 YARN 调度器未来性能的基石，但复杂度远超预期
3. **Fair Scheduler 的公平性在退化** — 2025-2026 年多个 Issue（YARN-11907 饥饿、YARN-11966 metrics 冲突）表明 FS 的基础公平性保障正在出现问题
4. **运维可观测性显著提升** — CS UI Phase 1/2（14 个子任务）是最大的新功能投入
